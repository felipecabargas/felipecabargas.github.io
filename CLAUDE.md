# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run build        # Build all articles from _articles/ → articles/*.html, update index.html and feed.xml
```

No test suite. No lint step. Build is the only script.

## Architecture

This is a static site hosted on GitHub Pages at `cabargas.com`. There is no framework — HTML, CSS, and vanilla JS only.

### Content Pipeline

Articles live as markdown in `_articles/<slug>/en.md` (required) and optionally `_articles/<slug>/es.md`. Running `npm run build` (`scripts/build.js`) compiles them to `articles/<slug>.html`, regenerates the article list in `index.html`, and writes `feed.xml`. The build script uses `gray-matter` for frontmatter and `marked` for markdown.

**CI (`build.yml`) triggers on pushes to `_articles/**`, `scripts/build.js`, or `package.json`.** It runs `npm run build` and commits the generated files back to master. This means: **never commit generated `articles/*.html`, `index.html`, or `feed.xml` manually after editing markdown** — CI handles it. For changes to static pages (`about.html`, `projects.html`, style), commit directly.

### Frontmatter spec (`en.md` required fields)

```yaml
title: "..."
date: "May 14, 2026"       # display date (English)
sortDate: "2026-05-14"     # ISO date for sort order
type: essay | note
topics: [ai, governance]   # lowercase, drives filter tabs on index
tags: [AI, Governance]     # display tags on article and index
excerpt: "..."
readtime: "3 min read"
slug: my-article-slug
originalLang: es            # optional — omit if article was written in English
```

For `es.md`, use the same fields except `slug` (not needed) and `topics` (not read from es.md); `date` should be the Spanish-formatted date string, `readtime` in Spanish.

### Bilingual system

**Article pages** (`articles/*.html`): generated by the build script. Two `<div class="lang-content" data-lang="en|es">` blocks, toggled by JS via `el.hidden`. The lang switcher only appears when an `es.md` exists.

**Static pages** (`index.html`, `about.html`, `projects.html`): hand-authored. Two patterns coexist:
- Inner-span approach (index): bilingual `<span data-lang="en|es">` inside shared structural elements — used on the article list so filter tabs (which operate on `.article-item` level) keep working.
- Full-section duplicate (about, projects): separate `<div data-lang="en">` and `<div data-lang="es" hidden>` blocks for each page's content.

**Critical**: The JS `apply()` function on static pages uses `[data-lang]:not(.lang-btn)` to avoid hiding the switcher buttons themselves (which also carry `data-lang` attributes).

Language detection order: `?lang=` URL param → `navigator.language` (auto-detects Spanish) → defaults to `en`. State is persisted via `history.replaceState` and `?lang=es` param.

### feed.xml merge conflicts

The CI workflow commits a new `lastBuildDate` timestamp in `feed.xml` every build. If you pull with `--rebase` after CI has run, `feed.xml` will always conflict. Resolve with:

```bash
git checkout --ours feed.xml
git add feed.xml
git rebase --continue
```

### Index article list

The `<!-- ARTICLES START --> ... <!-- ARTICLES END -->` block in `index.html` is fully replaced by the build script on every run. Do not hand-edit content inside these markers — it will be overwritten.

### `originalLang` flag

Set `originalLang: es` in `en.md` when the article was originally written in Spanish (e.g., `product-teams-losing-the-ai-race`). This causes the English article page to show "Originally written in Spanish. English translation by AI." and suppresses the notice on the Spanish side.
